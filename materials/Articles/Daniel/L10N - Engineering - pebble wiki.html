[ 1ISO codes ] [ 1.1iOS Language IDs ] [ 2URL template ] [ 3Boot server ] [ 4Translation Management ]

This page is to help coordinate localization implementation details across all impacted teams.

Lance Vick, Keegan Lillo, Steve Penna, Daniel Rodríguez Troitiño, Francois Baldassari, Monty Boyer, Kristen Culp

Note that this is related to Projects: Localization & Internationalization

Note that this is related to AppStore i18n service

 

ISO codes

The following ISO codes are already in use by the firmware team and will be used by the web, mobile and support teams. The support team has also begun work. If your team is not using these codes, please add a note to this page describing which code(s) your team are using in place of these:

  • fr_FR
  • en_US
  • de_DE
  • nl_NL
  • es_ES
  • zh_CN
  • zh_TW

iOS Language IDs

For completeness, iOS 8.0 seems to use this representations as the language IDs for the included languages:

  • English: en (→ en_US)
  • English (UK): en-GB (→ en_US)
  • English (Australia): en-AU (→ en_US)
  • English (India): en-IN (→ en_US)
  • Spanish: es (→ es_ES)
  • Spanish (Mexico): es-MX (→ es_ES)
  • French: fr (→ fr_FR)
  • French (Canada): fr-CA (→ fr_FR)
  • German: de (→ de_DE)
  • Chinese, Simplified: zh-Hans (→ zh_CN)
  • Chinese, Traditional: zh-Hant (→ zh_TW)
  • Chinese, Traditional (Honk Kong): zh-HK (→ zh_TW)
  • Dutch: nl (→ nl_NL)

(Even if it is incorrect to map a country language ID to another country, since our localizations doesn’t provide fallbacks, we will consider them fallbacks for its language component. The only one with a hardcoded transformation will be the Honk Kong one)

URL template

The web team is using the following URL format to serve localized views: 

http://app.pebble.com/{ISO_CODE}/rest/of#url

Boot server

The boot server will serve URLs depending on the app language, to do so the app will append a query string parameter named locale with the ISO locale ID of the app language (using underscore, language in small case, region in upper case). For example:

https://pebble-boot-staging.herokuapp.com/api/config/ios/v1/1/1?debug=true&locale=es_ES

If the locale parameter is missing or contains a value not supported by the server the default locale will be English (en_US).

Translation Management

We use POEditor (https://poeditor.com) for translation management of software projects. Contact poeditor@getpebble.com if you want a project created, or admin rights to an existing projects (credentials to the master account are shared via Meldium).

To use the POEditor API, you need the API Key: 19f6bcc92529e83c1b59a4f22d834f6a
